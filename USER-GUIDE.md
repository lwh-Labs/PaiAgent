## 快速入门

### 第一步：启动服务

1. **启动后端**
   ```bash
   cd backend
   mvn spring-boot:run
   ```

2. **启动前端**
   ```bash
   cd frontend
   npm run dev
   ```

3. **打开浏览器**
   访问 `http://localhost:5173`

### 第二步：创建工作流

1. 页面会自动创建一个默认工作流（包含 START 和 END 节点）
2. 从左侧节点面板拖入 LLM 节点和 TTS 节点到画板
3. 连接节点：START → LLM → TTS → END
4. 点击"保存"按钮保存工作流

### 第三步：配置节点

1. 点击 LLM 节点，配置 provider、model、apiKey
2. 点击 TTS 节点，配置 apiKey、voiceId
3. 点击"保存"保存配置

### 第四步：调试执行

1. 点击"调试"按钮打开调试抽屉
2. 在输入框中输入文字，如"请介绍一下人工智能"
3. 点击"运行工作流"
4. 等待执行完成，音频播放器将自动出现
5. 点击播放按钮收听 AI 播客

**操作**：拖拽节点到画板区域即可添加

**基础操作**：

| 操作 | 方式 |
|------|------|

**节点状态颜色**：

| 状态 | 颜色 | 说明 |

| 成功 | 绿色边框 | 执行成功 |
| 失败 | 红色边框 | 执行失败 |

## 创建第一个工作流

### 步骤 1：添加节点

1. 从左侧节点面板拖拽"大模型节点"到画板
2. 拖拽"超拟人音频合成节点"到画板

### 步骤 2：连接节点

1. 拖拽 START 节点底部的连接点到 LLM 节点顶部
2. 拖拽 LLM 节点底部的连接点到 TTS 节点顶部
3. 拖拽 TTS 节点底部的连接点到 END 节点顶部

连接完成后，工作流结构为：
```
START → LLM → TTS → END
```

### 步骤 3：配置 LLM 节点

1. 点击 LLM 节点，右侧弹出配置抽屉
2. 填写以下参数：
   - **Provider**：选择 LLM 提供商（OpenAI / DeepSeek / 通义千问）
   - **Model**：输入模型名称
     - OpenAI: `gpt-4o`, `gpt-4o-mini`, `gpt-3.5-turbo`
     - DeepSeek: `deepseek-chat`, `deepseek-reasoner`
     - 通义千问: `qwen-turbo`, `qwen-plus`, `qwen-max`
   - **API Key**：输入对应平台的 API 密钥
   - **System Prompt**（可选）：系统提示词，如"你是一个专业的播客主持人"
   - **Temperature**（可选）：生成温度，默认 0.7
   - **Max Tokens**（可选）：最大输出长度，默认 2048

3. 点击"保存配置"

### 步骤 4：配置 TTS 节点

1. 点击 TTS 节点，右侧弹出配置抽屉
2. 填写以下参数：
   - **API Key**：阿里云 DashScope API 密钥
   - **Voice ID**：音色 ID，如 `longxiaochun`
   - **Speed**（可选）：语速，默认 1.0
   - **Pitch**（可选）：音调，默认 1.0

3. 点击"保存配置"

### 步骤 5：保存工作流

1. 点击顶部工具栏的"保存"按钮
2. 工作流配置将保存到后端数据库

## 节点配置详解

### 用户输入节点（START）

- **类型**：工作流起始节点
- **输入**：无（由用户在调试抽屉中输入）
- **输出**：`{ text: "用户输入的文字" }`
- **配置**：无需配置

### 大模型节点（LLM）

- **类型**：文本处理节点
- **输入**：上游节点的文本输出
- **输出**：`{ text: "LLM 生成的文本" }`
- **配置项**：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| Provider | LLM 提供商 | openai / deepseek / qwen |
| Model | 模型名称 | gpt-4o, deepseek-chat, qwen-turbo |
| API Key | API 密钥 | sk-xxx... |
| System Prompt | 系统提示词 | 你是一个专业的播客主持人 |
| Temperature | 生成温度 | 0.7（范围 0~2） |
| Max Tokens | 最大输出长度 | 2048 |

**支持的模型**：

- **OpenAI**: gpt-4o, gpt-4o-mini, gpt-4-turbo, gpt-3.5-turbo
- **DeepSeek**: deepseek-chat, deepseek-reasoner
- **通义千问**: qwen-turbo, qwen-plus, qwen-max, qwen-long

### 超拟人音频合成节点（TTS）

- **类型**：语音合成节点
- **输入**：上游节点的文本输出
- **输出**：`{ audioUrl: "/api/audio/xxx.mp3", duration: 12.5 }`
- **配置项**：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| API Key | CosyVoice API 密钥 | sk-xxx... |
| Voice ID | 音色 ID | longxiaochun |
| Speed | 语速 | 1.0（范围 0.5~2.0） |
| Pitch | 音调 | 1.0（范围 0.5~2.0） |

**获取 CosyVoice API Key**：
1. 访问阿里云 DashScope 控制台
2. 创建 API Key
3. 确保已开通 CosyVoice 服务

### 结束节点（END）

- **类型**：工作流终止节点
- **输入**：上游节点的输出数据
- **输出**：将上游输出作为最终结果返回
- **配置**：无需配置

## 调试执行

### 打开调试抽屉

点击顶部工具栏的"调试"按钮，右侧弹出调试抽屉。

### 输入测试文字

在调试抽屉顶部的文本框中输入要测试的文字，例如：
```
请用通俗易懂的语言介绍一下人工智能的发展历程
```

### 执行工作流

点击"运行工作流"按钮，系统将：
1. 向后端发送执行请求
2. 后端按拓扑顺序依次执行每个节点
3. 实时更新画板上节点的执行状态
4. 在调试抽屉中展示每个节点的执行结果

### 查看执行结果

**节点结果卡片**：
- 节点名称和类型图标
- 执行状态标签（绿色=成功，红色=失败）
- 执行耗时
- 输入数据（可展开查看 JSON）
- 输出数据（可展开查看 JSON）
- 失败时显示错误信息

**执行流程示例**：
```
[START] 用户输入
  ↓ 输出: { text: "请介绍一下人工智能..." }
[LLM] 大模型处理
  ↓ 输出: { text: "人工智能的发展可以追溯到..." }
[TTS] 音频合成
  ↓ 输出: { audioUrl: "/api/audio/xxx.mp3", duration: 15.2 }
[END] 结束
  ↓ 最终结果: { audioUrl: "/api/audio/xxx.mp3" }
```

### 播放音频

当工作流执行成功且包含 TTS 节点时，调试抽屉底部会自动展示音频播放器：
- 点击播放按钮收听
- 拖拽进度条调整播放位置
- 调整音量大小

音频文件临时存储在后端，可直接通过播放器收听。

## 常见问题

### Q1: 后端启动失败，提示数据库连接错误

**原因**：MySQL 连接信息配置不正确

**解决**：
1. 检查 `backend/src/main/resources/application.yml` 中的数据库配置
2. 确保 MySQL 服务已启动
3. 确保数据库 `ai_agent_workflow` 已创建
4. 确保用户名密码正确

### Q2: 前端无法连接后端，提示跨域错误

**原因**：后端 CORS 配置问题或后端未启动

**解决**：
1. 确保后端服务已启动在 `http://localhost:8080`
2. 检查 `CorsConfig.java` 中是否允许了 `localhost:5173`

### Q3: LLM 节点执行失败

**原因**：
- API Key 无效或过期
- 模型名称不正确
- 网络连接问题

**解决**：
1. 检查 API Key 是否正确
2. 检查模型名称是否对应 Provider
3. 查看后端日志获取详细错误信息

### Q4: TTS 节点执行失败

**原因**：
- CosyVoice API Key 无效
- 未开通 CosyVoice 服务
- 文本过长或包含特殊字符

**解决**：
1. 检查阿里云 DashScope API Key
2. 在阿里云控制台确认已开通 CosyVoice 服务
3. 缩短输入文本长度

### Q5: 音频无法播放

**原因**：
- 音频文件未生成
- 音频 URL 不正确

**解决**：
1. 检查 TTS 节点是否执行成功
2. 检查后端日志确认音频文件已保存
3. 直接访问 `http://localhost:8080/api/audio/{filename}` 测试

### Q6: 工作流保存后，API Key 显示为 ****

**说明**：这是正常行为。API Key 在后端加密存储，返回给前端时自动脱敏为 `****`，不影响实际执行。

### Q7: 如何查看执行历史？

**方法**：
1. 后端提供了 API 接口 `GET /api/workflows/{id}/executions`
2. 可在浏览器直接访问查看 JSON 数据
3. 后续版本将在前端增加执行历史页面
